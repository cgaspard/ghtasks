## What's new in v0.1.8

### Fixes

- **Org projects now show up for team members.** Some teammates saw an empty **+ Project** dropdown even when they had access to organization projects. Root cause was an `orderBy: {field: TITLE}` clause on the `projectsV2` GraphQL query that GitHub was silently rejecting for certain org roles, nulling out the entire `organizations` branch. Dropped the server-side sort — projects are still sorted alphabetically client-side, so the UI is unchanged. If the dropdown was empty for you before, it should populate now.

### New

- **Developer Tools menu item.** The avatar popover now has a **Developer Tools** entry (between Settings and Sign out) that opens the WebView devtools in one click. Useful for inspecting network calls, reading console logs, and filing bug reports with actual diagnostics attached. Works in signed release builds.

### Diagnostics

- **Project picker empty-state hint.** If the project list comes back empty, the **+ Project** dialog now explains the three most common causes (OAuth app not approved by the org, SSO not authorized for the token, or a stale token that pre-dates a permission change) and points at the remedy for each.
- **`list_projects` console log.** The project loader now logs the project count and payload to the console so you can spot-check what GitHub returned without plumbing anything through the UI. Open **Developer Tools** to see it.

### Install

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized, universal — Apple Silicon + Intel)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — your sources, filters, and cached project data carry over.
