## What's new in v0.1.13

Update experience pass. The previous two releases added auto-updates, but the signal that an update is pending was easy to miss — users had to either click the avatar menu or open the About panel to find it. This release makes updates visible without hunting.

### New

- **Tray icon swaps when an update is available.** The menu-bar icon changes from the Octocat to a bold down-arrow-in-circle silhouette when a newer release is detected. The tooltip updates too: "GH Tasks — update to v0.1.14 available". Both render as native macOS template images, so they tint correctly for light and dark menu bars.
- **Bottom banner inside the window.** When an update exists, a compact amber strip appears at the bottom of the popover with the version, a one-click **Install** button, and a × to dismiss for the session. Clicking the version text opens the release notes on GitHub. The banner only renders when an update exists, so it costs no screen space most of the time.
- **Bigger, pulsing avatar dot.** The small orange dot that indicates an update is now 11px (was 9px) with a subtle 2-second pulse animation, making it readable at a glance. Still compact enough to not feel alarming.

### Fixes

- **About panel now propagates update state to the rest of the app.** Previously, clicking "Check again" in the About panel would find an update but leave the avatar dot / menu row / banner dark. All three now light up together whenever any code path (launch check, interval check, manual check) detects an update.
- **Background update check runs at 2 seconds instead of 5.** Shortens the window where a user could click their avatar before the launch check has completed.

### Install

This release is delivered automatically to anyone running v0.1.11+ — look for the update banner or the menu-bar icon change. Manual downloads:

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — nothing to reconfigure.
