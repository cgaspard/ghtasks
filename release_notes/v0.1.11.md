## What's new in v0.1.11

The big one: **auto-updates**. From this release forward, every time we publish a new version, your running copy picks it up silently in the background and prompts you to install. No more DMG downloads, no more nag messages.

### New

- **Auto-updater.** On launch (and every 4 hours while running) the app checks GitHub for a newer release. If one is found, an orange badge appears on your avatar and a **Update to v0.1.x** row shows at the top of the avatar menu. Click it to read the release notes and install in one click — the app downloads the signed bundle, verifies its signature against a public key baked into the app, replaces itself in place, and relaunches. Signed + notarized on macOS so Gatekeeper stays quiet. (*Users already on v0.1.10 or earlier need to install v0.1.11 manually; every release after this is automatic.*)
- **New About panel.** About has moved out of Settings into its own slide-forward panel, accessible from the avatar menu (or the version footer at the bottom of it). Shows the current version, a live **Check for updates** control, release-notes link for the running version, and a short "how it works" blurb explaining that your token lives in the OS keychain and your data never transits a third-party backend.
- **Fresh app icon.** New minimalist Dock / Finder / installer icon: a git-branch graphic with a green checkmark on the branched node. The familiar Octocat silhouette stays as the menu-bar tray icon.

### Security

Closed a cross-site-scripting vector in the issue-detail markdown rendering:

- **Rendered issue bodies and comments are now sanitized.** Previously we relied on the Content Security Policy to block inline `<script>` tags, but CSP does **not** block inline event handlers like `<img onerror="...">` or `javascript:` URIs. A malicious issue body could execute arbitrary code inside the webview, including calling any Tauri IPC command — which would let an attacker read your repo list, post comments as you, or create issues. All rendered markdown now passes through DOMPurify with a strict allowlist that refuses event handlers, non-http/https/mailto URIs, and script-adjacent tags.
- **Link clicks in rendered markdown validate the URL scheme before opening.** Even if something slipped past DOMPurify, we refuse to hand `file:`, `javascript:`, `data:`, `smb:`, or anything other than `http:` / `https:` / `mailto:` to the system opener.
- **Device-flow auth error paths now redact response bodies.** Defense in depth against a theoretical edge case where a parse failure could surface raw response bytes.

If you installed any prior version and have been viewing untrusted issues, there's no known in-the-wild exploit, but upgrading closes the vector.

### Fixes from v0.1.10 live testing

- **UI lockup when typing in the New Issue title.** The template-application effect was re-running on every keystroke and feeding back through the labels set. Fixed by isolating what the effect tracks.
- **Scroll wheel stopped working on the list panes after the slide-forward detail view was introduced.** The slide container's grid row was sizing to content, so the panes grew past the viewport and internal scroll regions had no bounded height. Fixed the grid sizing and made the pane height explicit.
- **Clicks could leak from the hidden list pane to the visible detail pane during the slide animation.** Added `inert` on whichever pane is off-screen.

### Install

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized, universal — Apple Silicon + Intel)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

This is the last release that requires a manual download for existing users. After you install v0.1.11, future updates happen automatically.
