## What's new in v0.1.10

This is a big one — the app goes from "list + create" to "read, triage, and discuss" without leaving the menu bar.

### New

- **Issue detail view.** Click the `›` arrow on any row in the Projects or Issues tab and the whole panel slides forward into a full-width reading view with the issue's rendered markdown body, every comment (avatars, author-association badges, relative timestamps), and an inline composer to post a new comment. The list pane is preserved underneath — tap **Back** (or press `Escape`) and you slide right back to exactly where you were, scroll position and filters intact. One level deep by design: `#123`-style cross-links in the body open on github.com rather than pushing another frame.
- **GitHub-flavored markdown rendering.** Issue bodies and comments render with headings, lists, tables, block + inline code, task lists, blockquotes, and embedded images from `user-images.githubusercontent.com`. Links intercept click and open in the system browser (never navigate the webview away).
- **Issue template support — markdown and YAML forms.** When you pick a repo in the **New Issue** modal, we fetch its `.github/ISSUE_TEMPLATE/` and show a **Template** dropdown with the same list GitHub shows you on the web. Includes a **Blank (no template)** default so you never have to use a template if you don't want to.
  - Markdown templates (`.md` with frontmatter) prefill title, body, and labels — without clobbering anything you've already typed.
  - YAML issue forms (`.yml` with input/textarea/dropdown/checkboxes fields) render as a real form inside the modal, validate required fields, and serialize on submit into the `### Label`-headed markdown body GitHub produces on the web.
  - Respects `config.yml`'s `blank_issues_enabled: false` if you've locked that down for your repo.

### Fixes

- **New Issue modal no longer locks the UI.** The template-application effect was re-running on every keystroke in the title input, creating a feedback loop through the labels set. Wrapped the effect body in `untrack` so it only runs when the template selection actually changes.
- **Scroll wheel works again on the list panes.** The forward/backward slide container was using a CSS grid whose rows sized to content, so the panes grew past the viewport and the internal `overflow: auto` regions had no bounded height to clip against. Fixed by pinning the grid row to 100% and setting an explicit `height: 100%` on each pane.
- **Inactive slide pane no longer steals clicks.** Added the `inert` attribute to whichever pane is off-screen so keyboard focus and pointer events stay scoped to the visible side.

### Under the hood

- Added `marked` (13kb gzipped) for markdown rendering, `serde_yaml` + `base64` on the Rust side for template parsing.
- New Rust commands: `list_issue_templates(repo)`, `get_issue_detail(repo, number)` — the latter fetches the full issue + all comments in parallel via `tokio::try_join!`.
- Templates are cached per-repo inside the New Issue modal so repeatedly flipping the repo dropdown doesn't re-hit the Contents API.
- Detail view has its own manual refresh button and `Escape`-to-back handling.

### Install

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized, universal — Apple Silicon + Intel)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — your sources, filters, and cached data carry over.
