## What's new in v0.1.14

Fixes "Launch at login" so it actually shows up where macOS users expect it.

### Fixes

- **Launch at login now appears in System Settings → General → Login Items.** The Settings toggle was wired to the autostart plugin's LaunchAgent mode, which writes a plist into `~/Library/LaunchAgents/` — that mechanism never appears in the macOS Login Items UI, so users toggling it on saw nothing change in System Settings and assumed it was broken. Switched to AppleScript mode, which registers the `.app` in the user's classic Login Items list (the one System Settings actually shows).
- **Cleans up stale LaunchAgent plists from prior versions.** If you toggled launch-at-login on under v0.1.13 or earlier, that LaunchAgent plist is automatically unloaded and removed at startup so you don't end up double-registered after this upgrade. Toggle the setting once after upgrading to apply your preference under the new mechanism.

### Install

This release is delivered automatically to anyone running v0.1.11+ — look for the update banner or the menu-bar icon change. Manual downloads:

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — nothing to reconfigure.
