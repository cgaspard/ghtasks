## What's new

**Themed dropdowns everywhere.** Every system `<select>` replaced with a dark-themed, keyboard-navigable `Select` component. Dropdowns now match the app's look, support arrow-key navigation and Enter to pick, and auto-enable a search box when a list has more than ~10 options (nice for repo pickers).

**Avatar menu.** The top bar is now just **Projects · Issues**. Click your avatar to open a popover with:

- **Settings** (also via `⌘,`)
- **Sign out**
- **Quit GH Tasks** (also via `⌘Q`)

The avatar popover shows your GitHub profile name + `@login` for quick identity confirmation.

**DevTools enabled in release builds.** Right-click inside the app window → **Inspect Element** to diagnose sync issues, inspect the token scopes, or capture console logs when troubleshooting with your team. This was only available in dev builds before.

## Why DevTools?

If your teammates can sign in but don't see shared projects in the `+ Project` dropdown, it's almost always one of:

- **Org hasn't approved the GH Tasks OAuth app.** Org owner: `https://github.com/organizations/<org>/settings/oauth_application_policy`
- **SSO authorization missing on their token.** User: `https://github.com/settings/applications` → **GH Tasks** → **Authorize** for the org
- **Token was minted before v0.1.3 added `read:org` scope.** Fix: sign out + sign in again.

## Installation

### macOS
Download **GH.Tasks_0.1.0_universal.dmg**, open it, drag **GH Tasks** to Applications. Signed + notarized — no Gatekeeper warning.

### Windows
Download the **x64-setup.exe** (recommended) or the **.msi** and run it.

### Linux
- **Debian / Ubuntu:** `GH.Tasks_0.1.0_amd64.deb` → `sudo dpkg -i`
- **AppImage:** `GH.Tasks_0.1.0_amd64.AppImage` → `chmod +x` → run
- **Fedora / RHEL:** `GH.Tasks-0.1.0-1.x86_64.rpm` → `sudo dnf install`

## Upgrading from v0.1.6

Settings, sources, filters, and window position all persist across upgrades. Nothing to reconfigure.
