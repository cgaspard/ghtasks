## What's new in v0.4.4

Surfaces it when macOS notification permission is off, instead of silently sending nothing.

### New

- **Notification-permission hint in Settings.** macOS never re-prompts once you've denied (or previously turned off) notifications for an app — until now GH Tasks had no way to tell you that was the reason nothing was arriving. Settings now shows a warning with a one-click link straight to System Settings → Notifications whenever permission is off.

### Install

This release is delivered automatically to anyone running v0.1.11+ — look for the update banner. Manual downloads:

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — nothing to reconfigure. If your desktop alerts stopped after upgrading to v0.4.3 or earlier, check Settings for the new notification warning, or go directly to System Settings → Notifications → GH Tasks and make sure it's allowed.
