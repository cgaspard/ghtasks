## What's new in v0.2.0

Filter your list by issue number, plus a batch of correctness fixes across the board, the New Issue modal, settings, and issue rendering.

### New

- **Filter by issue number.** The filter box on both the Projects and Issues tabs now understands numbers. Type `922` to jump to issue #922 (or anything whose number starts with `922`), or use `#922` for an exact match that ignores titles and labels entirely. A bare number still also matches titles and labels containing those digits, so nothing you could find before is lost. The placeholder now reads "Filter by title, label, or #number…" to make it discoverable.

### Fixes

- **Task-list checkboxes render again in issue bodies.** GitHub-flavored `- [x]` / `- [ ]` checklists in an issue's description and comments were being stripped out entirely by the HTML sanitizer. They now display correctly (still read-only), with no loss of the security hardening that strips everything else.
- **Issue-number search is precise.** `#50` now matches only issue #50, instead of also pulling in #150, #500, and every other number containing "50".
- **Settings can't be saved with a broken poll interval.** Clearing or mistyping the "Poll interval" field used to be able to send an invalid value to the backend. It's now always clamped to a valid whole number between 30 and 3600 seconds.
- **New Issue templates apply cleanly after switching repositories.** If you started typing a description, then changed the target repo and picked one of that repo's issue templates, the template's body is now filled in as expected instead of being suppressed by your earlier edit.
- **New Issue status is recorded against the right board.** When creating an issue in a project and switching projects while the request was in flight, the newly-created item could be attached to the wrong board's status. The choice is now locked in at submit time.
- **Stable list ordering.** Items with identical "last updated" timestamps now keep a deterministic order between refreshes instead of occasionally shuffling.
- **Custom-field filters no longer get stuck.** A "Filter by…" selection that referenced a field which is no longer visible (e.g. after removing the project that had it) now clears itself instead of silently doing nothing.
- **Project sync hardening.** Tightened the streaming-sync reconciliation so a refresh that overlaps an in-flight one can't drop items that are still on the board.

### Under the hood

- Added an automated QA suite (unit + end-to-end) that drives the real frontend in a headless browser against a mocked Tauri backend — runs with a single `npm test`, no GitHub auth or network required. This is contributor-facing only; it doesn't change the app.

### Install

This release is delivered automatically to anyone running v0.1.11+ — look for the update banner or the menu-bar icon change. Manual downloads:

- **macOS:** `GH.Tasks_universal.dmg` (signed + notarized)
- **Windows:** `GH.Tasks_x64-setup.exe` or `GH.Tasks_x64_en-US.msi`
- **Linux:** `GH.Tasks_amd64.AppImage`, `GH.Tasks_amd64.deb`, or `GH.Tasks-1.x86_64.rpm`

Upgrade in place — nothing to reconfigure.
