**GH Tasks** is a macOS / Windows / Linux menu-bar app that turns GitHub Issues and Projects v2 into a fast, keyboard-friendly task list. This is the first public release.

## Highlights

- **Projects v2 support, first-class.** Add a GitHub Projects v2 board as a source; items show up as cards with their Status, labels, assignees, and repo context.
- **Colored status picker on every card.** Click the Status pill to change a project item's status inline — the mutation runs server-side, the UI reflects it optimistically, and errors revert cleanly.
- **Repo sources too.** Add a repo + a GitHub search query (defaults to `is:issue is:open type:Task`). Pulls via the REST search API.
- **"Mine" view by default.** Both the Projects tab and the Issues tab default to showing items assigned to the signed-in user. Flip to "All" to see everyone's work.
- **Instant first paint.** Last-known data hydrates from disk cache on launch so you see your list in ~50ms while the fresh sync runs in the background.
- **Fast sync.** Server-side `-status:Released` filter, parallel-cursor pagination on refresh, and streamed pages bring full-snapshot fetches down from ~14s to ~4s on a 500-item board.
- **Create issues without leaving the app.** The New Issue modal supports:
  - Project mode (creates the issue + attaches it to the project + optionally sets Status — all in one click)
  - Repo-only mode
  - Multi-select labels pulled live from the target repo
  - "Assign to me" (on by default)
  - Type (defaults to Task)
- **Status-aware comment on any project item.** The 💬 button on a row expands an inline composer that posts a real GitHub comment.
- **Inline close confirmation on repo issues.** Click ○ and the card flips to a compact "Close this issue? Yes / No" strip — avoids destructive one-click accidents.

## How it's built

- **Tauri 2** (Rust + Svelte 5), native macOS menu-bar app.
- **GitHub OAuth device flow.** Token lives in the macOS Keychain; no client secret, no backend.
- **GraphQL + REST.** Projects v2 via GraphQL (with a tuned items query to keep per-page cost down), issue search + comment via REST.

## Installation

### macOS
1. Download `GH.Tasks_0.1.0_universal.dmg` (works on Apple Silicon + Intel).
2. Open the DMG and drag **GH Tasks** to Applications.
3. Launch from Applications. Signed + notarized — no Gatekeeper warning.

### Windows
1. Download `GH.Tasks_0.1.0_x64-setup.exe` (recommended) or the `.msi`.
2. Run the installer.

### Linux
- **Debian / Ubuntu:** download `GH.Tasks_0.1.0_amd64.deb` → `sudo dpkg -i`.
- **AppImage:** download `GH.Tasks_0.1.0_amd64.AppImage` → `chmod +x` → run.
- **Fedora / RHEL:** download `GH.Tasks-0.1.0-1.x86_64.rpm` → `sudo dnf install`.

## First-run setup

1. Launch the app. You'll land on a sign-in screen.
2. Click **Sign in with GitHub**. A browser opens to GitHub's device-code page; approve the requested scopes (`repo`, `read:org`, `read:user`, `notifications`, `project`).
3. Back in the app, add your first source:
   - **+ Project** for a GitHub Projects v2 board, or
   - **+ Repo** for a repo + search query.
4. Items stream into view. Use the filter pickers + Mine toggle to narrow.
